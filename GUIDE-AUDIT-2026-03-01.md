### 2. CSS 选择器重复定义（多处）
**问题描述**: `html` 和 `body` 的 `overflow-x:hidden` 被定义了两次：
- 第1次：全局样式开头（约第 38-39 行）

**代码位置**:
```css

/* 约 842-844 行 */

body{overflow-x:hidden;max-width:100vw}
```
**建议**: 删除媒体查询内的重复定义，保留全局样式即可。

### 3. `.ctn` 容器与导航栏宽度不一致（设计意图但需确认）
**问题描述**: 
- 主内容区 `.ctn` 使用 `max-width:880px`
- 导航栏 `.ni` 使用 `max-width:1200px`
- Hero 区域 `.hc` 使用 `padding:0 24px` 但没有 max-width 限制
**影响**: 导航栏比内容区宽，视觉上可能产生"内容收缩"感
**建议**: 这是设计选择，但如果希望视觉一致性，可考虑将 `.ni` 也限制为 880px 或与 `.ctn` 保持一致。

## 🟡 建议优化的地方

### 1. 响应式断点下的 section padding 递减可以更平滑
**当前**:
- 默认: `padding:96px 0`
- 768px: `padding:60px 0`
- 640px: `padding:48px 0`
- 480px: `padding:40px 0`

**建议**: 考虑在 1200px 断点添加一个中间值（如 80px）使过渡更平滑。

### 2. `.tw` 表格容器缺少 1200px 断点优化
**问题描述**: 表格容器 `.tw` 的 `min-width:820px` 在 1200px-900px 区间可能出现横向滚动条，而此时侧边导航刚隐藏。
**建议**: 在 1200px 断点添加表格字体大小调整，或考虑卡片式布局替代。

### 3. 移动端 `.sidenav` 隐藏但 DOM 仍存在
**问题描述**: 侧边导航在 ≤1200px 时 `display:none`，但元素仍在 DOM 中。
**建议**: 这是性能小优化点，可考虑使用 `visibility:hidden` 或完全移除，但当前实现也无功能问题。

### 4. `.cg` grid 布局 gap 在移动端未调整
**问题描述**: 默认 `gap:16px`，但在 480px 下未进一步减小。
**建议**: 在 480px 断点将 gap 减小到 10-12px 以最大化内容空间。

### 5. 颜色变量使用一致性
**发现**: 部分元素使用硬编码颜色值而非 CSS 变量：
- `#ef4444`（红色）在 `.fch .d1` 和 `.ct .n` 中使用
- `#eab308`（黄色）在 `.fch .d2` 中使用
- `#60a5fa`（蓝色）在 `.fl2 .re` 中使用
**建议**: 将这些颜色也定义为 CSS 变量以保持主题一致性。

### 6. `.hero-img` 在 768px 和 640px 断点有重复定义
**代码**:
```css
/* 768px */
.hero-img{max-width:220px;margin-bottom:28px}

/* 640px 无覆盖 */

/* 480px */
.hero-img{max-width:180px;margin-bottom:20px}
```
**建议**: 640px 断点可考虑添加中间值（如 200px）使过渡更自然。
