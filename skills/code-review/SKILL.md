# Code Review Skill — allen00.top 专用

针对个人静态网站（纯 HTML/CSS/JS）的代码审查规范。不涉及后端、数据库、测试框架。

---

## 审查优先级

| 维度 | 重点 | 优先级 |
|------|------|--------|
| 视觉正确性 | 页面在不同设备上显示是否正常 | 高 |
| SEO | meta 标签、结构化标题、sitemap | 高 |
| 性能 | 图片大小、资源加载顺序 | 高 |
| 可访问性 | alt 文本、语义化标签、对比度 | 中 |
| 代码可维护性 | 结构清晰、命名一致、无冗余 | 中 |
| 安全 | 外链处理、无敏感信息泄露 | 中 |

---

## 检查清单

### 1. SEO 检查（每篇文章必做，用 seo-audit skill 辅助）
- [ ] `<title>` 存在，25-35 中文字，结尾含 "- Allen"
- [ ] `<meta name="description">` 存在，70-120 字
- [ ] `<link rel="canonical">` 指向正确 URL，末尾有 `/`
- [ ] og:title / og:description / og:image / og:url 完整
- [ ] 每页只有一个 `<h1>`，与 title 主题一致
- [ ] heading 层级不跳级（h1 → h2 → h3）
- [ ] sitemap.xml 已更新（用 seo-sitemap skill）

### 2. 性能检查
- [ ] 图片单张不超过 300KB
- [ ] 图片使用 `loading="lazy"`（首屏图除外）
- [ ] CSS/JS 放在正确位置（CSS 在 `<head>`，JS 在 `</body>` 前）
- [ ] 没有引入用不上的大型库
- [ ] 字体加载有 `font-display: swap`

### 3. 响应式 & 移动端
- [ ] `<meta name="viewport">` 存在
- [ ] 在 375px 宽度下页面正常显示，无横向滚动
- [ ] 表格在小屏有横向滚动或改为卡片布局
- [ ] 导航在手机端有汉堡菜单，菜单背景完全不透明
- [ ] 字体大小在移动端不低于 14px

### 4. 可访问性
- [ ] 所有 `<img>` 有 `alt` 属性（纯装饰图用 `alt=""`）
- [ ] 链接文字有描述性，不用"点击这里"
- [ ] 颜色对比度足够（正文文字对背景对比度 ≥ 4.5:1）
- [ ] 交互元素可以用键盘访问

### 5. 代码规范
- [ ] HTML 结构语义化（用 `<article>`、`<nav>`、`<main>` 等）
- [ ] CSS 类名命名一致（与现有风格保持一致）
- [ ] 没有内联 style（颜色、间距等用 CSS 变量）
- [ ] 没有注释掉的死代码
- [ ] 没有重复的 `<meta>` 标签

### 6. 安全
- [ ] 外链加 `rel="noopener noreferrer"`
- [ ] 没有暴露个人信息（邮箱、电话等只在 about 页）
- [ ] 没有硬编码的 API key 或 token

---

## 文章页专项检查

新文章上线前额外检查：

- [ ] og:image 图片文件实际存在
- [ ] 文章内有至少一个内链（指向其他文章或页面）
- [ ] 发布日期正确
- [ ] draft 文件已从 `drafts/` 移出或标记为已发布
- [ ] `blog/index.html` 文章列表已更新

---

## 三步审查流程

**第一步（2分钟）：整体扫描**
- 页面结构和视觉是否正常
- 文件放对位置了吗
- 有没有明显遗漏

**第二步（主要时间）：逐项检查**
- 按上方清单逐条过
- 在手机模拟器（Chrome DevTools）预览一遍

**第三步（2分钟）：SEO 收尾**
- 运行 seo-audit 检查脚本
- 确认 sitemap 已更新

## 严重程度

| 级别 | 含义 | 是否阻止发布 |
|------|------|------------|
| `[阻塞]` | SEO 严重缺失、页面无法正常显示、敏感信息泄露 | 是 |
| `[重要]` | 移动端显示异常、图片过大、可访问性缺失 | 建议修复后发布 |
| `[优化]` | 代码冗余、命名不规范、可改进但不影响使用 | 否 |
